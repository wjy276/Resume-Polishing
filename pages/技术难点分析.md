### 3.8 流式输出对接 ★★★★☆

**难度评估：较难**

#### 功能需求
- AI 生成内容流式展示
- 简历优化流式输出
- 模拟面试对话流式输出
- 进度提示与取消

#### 技术要点
| 要点 | 说明 |
|------|------|
| SSE 对接 | EventSource 或 fetch + ReadableStream |
| WebSocket | 实时双向通信 |
| 流式渲染 | 逐字/逐段渲染，避免卡顿 |
| 错误重连 | 网络断开重连机制 |
| 取消请求 | AbortController 中断请求 |

#### 实现建议
```javascript
// SSE 流式请求示例
const fetchStream = async (url, onMessage) => {
  const response = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  })

  const reader = response.body.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break

    const chunk = decoder.decode(value)
    onMessage(chunk)
  }
}

// uni-app 中使用 uni.request + responseType
```

#### 注意事项
- uni-app H5 端支持 SSE/WebSocket
- 小程序端需使用 `uni.request` 或 `uni.connectSocket`
- 需处理跨端兼容性

#### 工作量估算
- 开发时间：5-7 天
- 依赖：智能体流式接口

### 6.2 关键路径

```
登录认证 → 简历上传 → 流式输出 → 模拟面试 → 评估报告
   ↓           ↓          ↓          ↓          ↓
 用户系统    文件处理    AI 对接    对话系统    可视化
```

### 6.3 建议开发策略

1. **先易后难**：从基础 UI 开始，逐步增加复杂度
2. **接口先行**：与后端约定接口规范，Mock 数据并行开发
3. **模块化开发**：每个功能模块独立，便于测试和维护
4. **增量迭代**：优先完成 P0 功能，逐步完善 P1/P2
